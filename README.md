# Telegram Бот для учета заказов кофе

Система для управления заказами кофе с интеграцией Google Sheets и развертыванием на Koyeb.com

## 🎯 Возможности

### Для всех пользователей:
- ✅ Оформление новых заказов
- ✅ Просмотр статистики (день/месяц)
- ✅ Поиск торговых точек

### Для менеджеров:
- ✅ Все возможности выше
- ✅ Добавление расходов
- ✅ Корректировка количества товара
- ✅ Изменение стоимости товара

### Для владельцев и разработчиков:
- ✅ Все возможности выше
- ✅ Управление адресами (добавление, редактирование, удаление)
- ✅ Управление товарами (добавление, редактирование, удаление)
- ✅ Управление остатками
- ✅ Контроль минимальных остатков

## 📋 Предварительные требования

1. Аккаунт на [Telegram](https://telegram.org)
2. Аккаунт на [Google Cloud Platform](https://console.cloud.google.com)
3. Аккаунт на [Koyeb](https://app.koyeb.com)
4. Аккаунт на [GitHub](https://github.com) (опционально, но рекомендуется)

## 🚀 Шаг 1: Создание Telegram бота

1. Откройте Telegram и найдите бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Введите имя бота (например, "Coffee Orders Bot")
4. Введите username бота (например, "coffee_orders_bot")
5. Сохраните полученный токен (например: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

### Получение ID пользователя:

1. Найдите бота [@userinfobot](https://t.me/userinfobot)
2. Отправьте команду `/start`
3. Бот пришлет ваш ID (например: `123456789`)

## 🗂️ Шаг 2: Настройка Google Sheets

### Создание проекта в Google Cloud:

1. Перейдите на [Google Cloud Console](https://console.cloud.google.com)
2. Создайте новый проект или выберите существующий
3. В меню навигации выберите **APIs & Services** → **Library**
4. Найдите и включите **Google Sheets API**
5. Найдите и включите **Google Drive API**

### Создание Service Account:

1. Перейдите в **APIs & Services** → **Credentials**
2. Нажмите **Create Credentials** → **Service Account**
3. Введите имя (например, "coffee-bot-service")
4. Нажмите **Create and Continue**
5. В разделе **Role** выберите **Editor**
6. Нажмите **Continue** → **Done**

### Получение JSON ключа:

1. Нажмите на созданный Service Account
2. Перейдите на вкладку **Keys**
3. Нажмите **Add Key** → **Create new key**
4. Выберите формат **JSON**
5. Сохраните скачанный файл

### Создание Google таблицы:

1. Откройте [Google Sheets](https://sheets.google.com)
2. Создайте новую таблицу
3. Скопируйте ID таблицы из URL:
   - URL: `https://docs.google.com/spreadsheets/d/1ABC123xyz/edit`
   - ID: `1ABC123xyz`
4. Откройте JSON файл с ключом Service Account
5. Найдите поле `"client_email"` (например: `coffee-bot@project.iam.gserviceaccount.com`)
6. В Google таблице нажмите **Поделиться**
7. Добавьте email из `client_email` с правами **Редактор**

## 🔧 Шаг 3: Настройка проекта

### Структура проекта:

```
coffee-bot/
├── bot.py              # Основной код бота
├── requirements.txt    # Зависимости Python
├── Dockerfile          # Конфигурация Docker
├── .gitignore         # Игнорируемые файлы
├── .env.example       # Пример переменных окружения
└── README.md          # Документация
```

### Настройка переменных окружения:

Создайте файл `.env` (локально, для тестирования):

```env
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
GOOGLE_SHEET_ID=1ABC123xyz
GOOGLE_CREDENTIALS={"type":"service_account","project_id":"your-project",...}
BOT_USERS={"123456789":"developer","987654321":"owner","555555555":"manager"}
```

**Важно:** 
- `GOOGLE_CREDENTIALS` должен быть содержимым JSON файла в одну строку
- `BOT_USERS` содержит ID пользователей и их роли
- Роли: `developer`, `owner`, `manager`

## 📦 Шаг 4: Развертывание на Koyeb

### Подготовка кода:

1. Создайте репозиторий на GitHub (или используйте Docker Hub)
2. Загрузите все файлы проекта (кроме `.env`)
3. Убедитесь, что `.gitignore` настроен правильно

### Развертывание:

1. Войдите на [Koyeb](https://app.koyeb.com)
2. Нажмите **Create App**
3. Выберите **GitHub** или **Docker**

#### Вариант A: Через GitHub:
- Подключите свой репозиторий
- Выберите ветку (обычно `main`)
- Koyeb автоматически обнаружит Dockerfile

#### Вариант B: Через Docker Hub:
- Постройте образ локально: `docker build -t username/coffee-bot .`
- Загрузите образ: `docker push username/coffee-bot`
- Укажите `username/coffee-bot` в Koyeb

### Настройка переменных окружения в Koyeb:

В разделе **Environment variables** добавьте:

1. **TELEGRAM_BOT_TOKEN**
   - Value: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`

2. **GOOGLE_SHEET_ID**
   - Value: `1ABC123xyz`

3. **GOOGLE_CREDENTIALS**
   - Value: Весь JSON в одну строку (без переносов)
   - Пример: `{"type":"service_account","project_id":"my-project-123",...}`

4. **BOT_USERS**
   - Value: `{"123456789":"developer","987654321":"owner","555555555":"manager"}`

### Настройка развертывания:

1. **Instance Type:** Free (или Nano)
2. **Regions:** Выберите ближайший регион
3. **Port:** Оставьте пустым (бот не использует HTTP)
4. **Health checks:** Отключите (бот не отвечает на HTTP)

### Запуск:

1. Нажмите **Deploy**
2. Дождитесь завершения развертывания
3. Проверьте логи на наличие ошибок

## 🔍 Устранение проблем

### Бот не отвечает:

1. Проверьте логи в Koyeb
2. Убедитесь, что `TELEGRAM_BOT_TOKEN` правильный
3. Проверьте, что у бота есть разрешение писать пользователям

### Ошибки Google Sheets:

1. Убедитесь, что Service Account email добавлен в таблицу
2. Проверьте, что APIs включены в Google Cloud
3. Проверьте формат `GOOGLE_CREDENTIALS` (должен быть JSON в одну строку)

### Нет доступа к боту:

1. Проверьте, что ваш ID добавлен в `BOT_USERS`
2. Убедитесь, что роль указана правильно
3. Перезапустите бота в Koyeb

### Проблемы с развертыванием:

1. Проверьте, что все файлы загружены
2. Убедитесь, что Dockerfile правильный
3. Проверьте логи сборки в Koyeb
